# Data Format

## Current dataset artifacts

The current local training snapshot is generated by the HuggingFace pipeline in
`download_hf_contracts.py` and consumed by `train.py`. These `data/` artifacts
are generated and gitignored, so a clean clone must regenerate or restore them:

| Artifact | Purpose | Current local rows |
|---|---|---:|
| `data/hf_training_dataset.jsonl` | Full supervised source dataset | 31,577 |
| `data/hf_training_dataset.jsonl.rejects.jsonl` | Export-time quarantined rows | 4,728 |
| `data/train_dataset.jsonl` | Leakage-free training split | 25,723 |
| `data/val_dataset.jsonl` | Validation split | 3,964 |
| `data/test_dataset.jsonl` | Holdout/test split | 1,890 |
| `data/split_manifest.json` | Split lineage, leakage checks, coverage, artifact hashes | n/a |
| `data/manifests/hf_*_manifest.json` | Download, compile, and export provenance | n/a |
| `data/preflight_cache/*.preflight*.json` | Cached schema/token-length preflight reports | n/a |

The Etherscan `DatasetBuilder` path writes the same row schema to
`data/smart_contract_dataset.jsonl` by default, with
`data/smart_contract_dataset.jsonl.rejects.jsonl` and optional
`data/smart_contract_dataset.partial.jsonl` quarantine artifacts.

## Training JSONL schema

Each non-empty JSONL line is one supervised TAC-to-Solidity example:

```json
{
  "input": "function selector_a9059cbb:\n  // Selector: 0xa9059cbb\n  // Entry block: block_0080\n  ...",
  "output": "function transfer(address to, uint256 amount) public returns (bool) { ... }",
  "metadata": {
    "schema_version": 1,
    "function_name": "transfer",
    "function_signature": "function transfer(address to, uint256 amount)",
    "selector": "0xa9059cbb",
    "visibility": "public",
    "is_payable": false,
    "is_view": false,
    "contract_address": "0x...",
    "contract_name": "Token",
    "target_contract_name": "Token",
    "compiled_contract": "Token",
    "compiler_version": "0.8.20",
    "optimizer_enabled": true,
    "optimizer_runs": 200,
    "source": "huggingface",
    "body_hash": "...",
    "input_hash": "...",
    "output_hash": "...",
    "final_row_hash": "...",
    "quality": {
      "input_token_estimate": 128,
      "output_token_estimate": 96,
      "tac_instruction_lines": 42
    }
  }
}
```

Required fields are non-empty string `input` and `output`. `metadata` must be an
object for schema v1 rows and must include `metadata.schema_version: 1` unless a
legacy preflight is explicitly allowed with `--allow-legacy-metadata-schema`.

### Prompt safety

`input` is sanitized, bytecode-only TAC. Source/oracle annotations such as
compiler comments, optimizer settings, Solidity signatures, ABI parameter names,
returns, and source-derived storage-layout comments are removed from prompt text.
Compiler/source facts may remain in `metadata` for filtering, analysis,
manifests, and evaluation segmentation, but are not inserted into prompts.

During training, `SmartContractDataset` uses the Alpaca template by default:

```text
### Instruction:
Convert the following Three-Address Code (TAC) representation to readable Solidity code.

### Input:
Bytecode metadata: selector=0xa9059cbb, tac_blocks=..., tac_ops=...

<input TAC>

### Response:
<output Solidity>
```

The optional metadata line is derived only from bytecode/TAC evidence: selector,
TAC block/op counts, branch/storage/call/log/revert counts, and optional bytecode
length/instruction/function counts. Labels mask the prompt prefix with `-100`, so
only the response target (and simple-template suffix, when used) contributes to
loss. If context must be truncated, the tokenizer keeps supervised target tokens
first and truncates TAC/prompt budget before dropping the target.

## Metadata keys

| Key group | Keys | Notes |
|---|---|---|
| Schema | `schema_version` | Current value is `1`. |
| Function identity | `function_name`, `function_signature`, `selector`/`function_selector`, `visibility`, `is_payable`, `is_view` | Used for reporting and split leakage keys. |
| Contract identity | `contract_address` or `contract_id`, `contract_name`, `target_contract_name`, `compiled_contract`, `source` | `contract_address` must be a 20-byte `0x` hex string when present. HF rows use `source: huggingface`. |
| Compiler facts | `compiler_version`, `optimizer_enabled`, `optimizer_runs` | Retained for stratification and analysis only; never prompt text. |
| Dedup/decontamination | `source_hash`, `source_code_hash`, `body_hash`, `input_hash`, `output_hash`, `pair_norm_hash`, `tac_hash`, `final_row_hash` | Exporters compute exact `input_hash`, `output_hash`, and `final_row_hash` after prompt sanitization. Normalized hashes are 32-hex digests; source/exact/final hashes are commonly SHA-256. |
| Bytecode/eval evidence | `runtime_bytecode`, `deployed_bytecode`, `opcodes`, `opcode_sequence`, `bytecode_analysis`, `bytecode_metadata`, `differential_calls` | Optional fields consumed by bytecode-grounded evaluation when present. |
| Quality instrumentation | `quality.input_chars`, `quality.output_chars`, `quality.input_lines`, `quality.output_lines`, `quality.input_token_estimate`, `quality.output_token_estimate`, `quality.tac_instruction_lines` | Approximate, tokenizer-independent signals emitted on accepted rows and rejects for automated data triage. |
| Quarantine markers | `partial`, `partial_split` | Partial placeholders are excluded from the default supervised dataset. |

The shared schema validator checks critical fields: address format, selector
format, hash-like keys (`source_hash`, `source_code_hash`, `body_hash`, `input_hash`,
`output_hash`, `pair_norm_hash`, `tac_hash`, `final_row_hash`), boolean
metadata, `compiler_version` semver shape, and non-negative integer
`optimizer_runs`.

## TAC format

Three-Address Code is emitted by `BytecodeAnalyzer` per bytecode function:

```text
function selector_a9059cbb:
  // Selector: 0xa9059cbb
  // Entry block: block_0x0080
  block_0x0080:
    // Successors: block_0x0095, block_0x00a0
    temp_1 = CALLDATALOAD 0x04
    temp_2 = CALLDATALOAD 0x24
    temp_3 = ISZERO temp_1
    COND_JUMP temp_3 -> block_0x00a0
  block_0x0095:
    temp_4 = SLOAD storage[temp_1]
    temp_5 = SUB temp_4, temp_2
    SSTORE storage[temp_1], temp_5
    RETURN 0x01
  block_0x00a0:
    REVERT
```

Instruction categories include arithmetic (`ADD`, `SUB`, `MUL`), comparisons
(`LT`, `GT`, `EQ`, `ISZERO`), bitwise ops (`AND`, `OR`, `SHL`, `SHR`), memory
and calldata (`MLOAD`, `MSTORE`, `CALLDATALOAD`), storage (`SLOAD`, `SSTORE`),
control flow (`JUMP`, `COND_JUMP`, `RETURN`, `REVERT`, `STOP`), calls (`CALL`,
`STATICCALL`, `DELEGATECALL`, `CREATE`), logs (`LOG0`-`LOG4`), and environment
ops (`CALLER`, `CALLVALUE`, `ADDRESS`, `TIMESTAMP`, `GAS`).

## Shared export primitives

`src.dataset_export_primitives` is the source of truth used by both export paths:

| Primitive | Purpose |
|---|---|
| `sanitize_tac_prompt_input` | Removes source/compiler-only prompt annotations and rewrites unsafe function headers. |
| `build_training_record` | Produces `{input, output, metadata}` and injects schema v1 metadata. |
| `hash_normalized_body`, `hash_normalized_tac`, `hash_normalized_pair` | Stable deduplication hashes. |
| `record_final_row_hash` / `final_row_hash` | Detect exact final `input`+`output` duplicates. |
| `tac_quality_reject_reasons` | Flags TAC extraction errors such as stack underflow, unresolved jumps, and extraction failures. |
| `auxiliary_contract_reject_reasons` | Quarantines compiled auxiliary-contract rows when they do not match the target contract. |
| `export_length_report` | Estimates context/target tokens using the same prompt shape used for training preflight. |
| `validate_training_record_schema` | Validates row and critical metadata schema. |

The HF exporter can also validate existing JSONL with:

```bash
uv run python download_hf_contracts.py --validate-jsonl data/hf_training_dataset.jsonl
```

## Split manifest

`train.py` creates or reuses `data/split_manifest.json` when splitting a source
JSONL. The manifest includes:

- `manifest_kind: dataset_split`, `schema_version: 2`, `created_at`, and source
  dataset SHA-256;
- split parameters: train/val/test ratios (`0.85/0.10/0.05` by default), seed,
  group-key precedence, leakage categories, stratification fields, and split
  quality gates;
- row and group counts, output artifact paths, sizes, hashes, and row counts;
- `leakage_validation` over `source_hash`, `contract_address`,
  `contract_function`, `body_hash`, exact `input_hash`, and exact `output_hash`;
- holdout `coverage` over compiler version, optimizer, visibility, source,
  length bucket, and function family;
- `split_quality` target-vs-actual counts and oversized-component checks;
- cache status when `--reuse-splits` is used.

Splits are leakage-connected components, not independent rows: rows sharing any
leakage key stay in the same split. If fewer than three groups exist, all rows
are written to train and validation/test are left empty to avoid leakage.

## Quality checks and rejects

Export-time rejects are written to `<output>.rejects.jsonl`. Reject rows include
source row number, contract/function identifiers, `body_hash`, `final_row_hash`,
optional `compiled_contract`/`target_contract_name`, token-length diagnostics,
reject reasons, and duplicate provenance when applicable.

| Check | Reject/manifest signal |
|---|---|
| Normalized body frequency cap | `validation.body_duplicate_cap`, `drop_counts.body_cap_rows` |
| Exact final `input`+`output` duplicate | `final_row_duplicate`, `validation.final_row_duplicates` |
| TAC extraction quality | `tac_empty`, `tac_no_operations`, `tac_stack_underflow`, `tac_extraction_error`, `tac_analysis_error`, `tac_unresolved_jump` |
| Prompt/source leakage | `prompt_source_metadata_leak`, `prompt_source_signature_leak`, `selector_input_mismatch` |
| Completion/schema quality | `output_empty`, `output_missing_solidity_entrypoint`, `partial_placeholder`, `metadata_schema_invalid` |
| Auxiliary compiled contract | `auxiliary_compiled_contract` |
| Token budget | `context_overlength`, `target_overlength`, `validation.token_length_filter` |
| Partial placeholders | `partial_placeholder`, optional `smart_contract_dataset.partial.jsonl` |

Both export paths default export-time token filtering to an 8192-token maximum
sequence length. Lower budgets can still be supplied explicitly for ablation or
smoke-test runs.

`train.py` runs data preflight before training and eval-only runs unless
`--skip-data-preflight` is set. Preflight validates JSON parsing, row type,
required fields, metadata schema, and tokenizer-aware target/context length. It
uses local tokenizer files by default, can download with
`--preflight-tokenizer-download`, can explicitly fall back to approximate
whitespace counts with `--allow-whitespace-preflight-fallback`, and caches
reports under `data/preflight_cache/` by default.

## SQLite database schema

`data/contracts.db` supports both HuggingFace and Etherscan generation paths.
Important tables:

| Table | Key columns | Purpose |
|---|---|---|
| `contracts` | `address`, `source_code`, `bytecode`, `compiler_version`, `optimization_enabled`, `optimization_runs`, `abi`, `contract_name`, `source`, `source_hash`, `processed`, `compile_status`, `attempt_count`, `last_error` | Contract source/runtime-bytecode cache and compile status. |
| `function_pairs` | `contract_address`, `function_name`, `tac_representation`, `solidity_code`, `function_signature`, `visibility`, `is_payable`, `is_view`, `metadata`, `hash`, `body_hash`, `tac_hash`, `pair_norm_hash` | Stored TAC↔Solidity pairs before export selection. |
| `selector_registry` | `selector`, `signature`, `source`, `occurrences` | Selector-to-signature registry seeded with built-ins and learned from compiled contracts. |
| `compile_diagnostics` | `run_id`, `contract_address`, `compiler_version`, `optimizer_enabled`, `optimization_runs`, `status`, `error` | HF compile/analysis diagnostics. |
| `generation_diagnostics` | `run_id`, `stage`, `contract_address`, compiler fields, `contract_name`, `status`, `error` | Etherscan collection/generation diagnostics. |
| `dataset_filter_drops` | `run_id`, `rule`, `rows_dropped` | Etherscan filter drop accounting. |

## Manifests

`download_hf_contracts.py --manifest-dir data/manifests` writes:

- `hf_download_manifest.json`: HuggingFace lineage (`andstor/smart_contracts`,
  config `flattened`, split `train`, requested/resolved revision), streamed
  Parquet parameters/performance, artifact hashes, row/status/drop counts.
- `hf_compile_manifest.json`: compile parameters (`max_compiler_versions`,
  optimizer variants, workers, body cap), row counts, status/drop counts, and
  summarized compile/analysis failures.
- `hf_export_manifest.json`: output/reject/database artifacts, export selection,
  duplicate stats, validation blocks, reject counts, solc versions, and
  `training_row_schema_version`.

The Etherscan `DatasetBuilder.export_dataset()` path writes
`smart_contract_dataset.<format>.manifest.json` with command/git/environment
metadata, input address-list lineage, dataset/filter diagnostics, artifacts,
reject counts, and training row schema version.
